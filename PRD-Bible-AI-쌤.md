# PRD: BibleSsam

> **앱 이름**: BibleSsam (바이블쌤)  
> **한 줄 정의**: **지금 내 상황에 맞는 말씀**을 찾아 주고, AI가 **쉽게 설명 + 삶에 적용**까지 도와주는 성경 도우미(웹 + 앱).

> **문서 목적**: 제품 범위·기능·플랫폼·API 선정·기술 스택을 정리하고, MVP 및 이후 단계를 정의한다.

---

## 0. 한눈에 보는 결론

- **제품 한 문장**: **"지금 내 상황에 딱 맞는 말씀"**을 보여 주고, AI가 **쉬운 설명 + 내 삶에 적용하는 한 줄**까지 제공하는 성경 도우미.
- **읽기 단위**: **1장(Chapter, 약 20~30절)** — 매일 1장을 읽고, AI가 해당 장 전체를 쉽게 설명해 준다.
- **핵심 차별점 (vs YouVersion, 갓피플 등)**:
  1. **AI 쉬운 설명 + 삶 적용 한 줄** — 기존 성경 앱에 없는 기능. 장 전체를 쉽게 풀어주고, 삶에 적용하는 한 줄까지.
  2. **감정·상황 기반 추천** — "위로가 필요할 때", "불안할 때" 등 내 상태에서 출발 → 기존 앱의 "책/장/절" 탐색과 근본적으로 다른 진입점.
  3. **읽기 스트릭 (10분 기준)** — 매일 10분 이상 말씀을 읽으면 연속 기록이 쌓여 습관화를 돕는다.
- **플랫폼**: **웹** + **Flutter 앱**(iOS/Android).
- **핵심 기능**: (1) 오늘의 말씀 (2) AI 쉬운 설명 + 삶 적용 (3) 감정·상황 맞춤 추천 (4) 읽기 스트릭 (5) 즐겨찾기 (6) 오늘의 한 줄 기도 (7) 검색 (8) 전체 목록.
- **UI 구조**: 하단 3탭 — **홈** | **즐겨찾기** | **설정**.
- **성경 데이터**: 무료 API 우선 사용. 한글은 API 제한이 있으므로 단계별 대응.
- **AI API**: 무료 티어 LLM API 활용 (Google Gemini Free / OpenRouter Free / Groq Free).

## 1. 배경 / 문제 정의 (Why)

### 경쟁 환경

| 앱 | 사용자 | 강점 | 약점 |
|----|--------|------|------|
| **YouVersion** | 5억+ 다운로드 | 읽기 플랜, 다국어, 오늘의 구절 | AI 설명 없음, 적용 도움 없음 |
| **갓피플 성경** | 한국 시장 1위 | 한글 성경, QT, 검색 | AI 설명 없음, 감정 기반 추천 없음 |
| **대한성서공회** | 공식 성경 | 한글 원문 정확 | 기능 최소, UX 낡음 |

**기존 앱들이 해결하지 못하는 것**:
- 읽어도 **무슨 뜻인지 모르겠다** → 쉬운 설명이 없다.
- 읽어도 **내 삶에 어떻게 적용하는지 모르겠다** → "그래서 나는 뭘 하라는 건데?"에 대한 답이 없다.
- **지금 힘든 상황에 맞는 말씀**이 뭔지 모르겠다 → 책/장/절로만 탐색해야 한다.
- 혼자 읽으니 **꾸준히 안 된다** → 습관화 장치가 약하다.

### 페인포인트 정의

| # | 페인포인트 | 본질 | BibleSsam의 해결책 |
|---|-----------|------|---------------------|
| 1 | "읽어도 **무슨 뜻인지** 모르겠다" | 이해 장벽 | **AI 쉬운 설명** (장 전체의 핵심 내용·배경·문맥을 쉽게 풀어줌) |
| 2 | "읽어도 **내 삶에 어떻게 적용**하는지 모르겠다" | 적용 장벽 | **AI 삶 적용 한 줄** ("오늘 이렇게 적용해 보세요") |
| 3 | "**지금 내 상황**에 맞는 말씀이 뭔지 모르겠다" | 진입점 부재 | **감정·상황 맞춤 추천** (위로/감사/불안/용기 등에서 출발) |
| 4 | "혼자라서 **꾸준히 안 된다**" | 습관화 실패 | **읽기 스트릭** (매일 10분 읽기 + 연속 일수 기록) |

## 2. 제품 정의 (What)

BibleSsam은 **"내 상황에 맞는 말씀 + AI 맞춤 설명"** 서비스다.  
기존 성경 앱이 "성경 텍스트를 보여주는 도구"라면, BibleSsam은 **"성경을 이해하고 삶에 적용하도록 돕는 AI 도우미"**다.

### 설계 원칙

- **장(Chapter) 단위 읽기**: 기본 읽기 단위는 1장(약 20~30절). 한 구절만 보는 것이 아니라 문맥 있는 분량을 읽는다.
- **삶 적용 중심**: AI는 "해석"이 아니라 **쉬운 설명 + "오늘 내 삶에 적용하면"** 한 줄을 제공.
- **감정·상황 진입**: 사용자는 책/장/절이 아니라 **"지금 내 감정/상황"**에서 출발할 수 있다.
- **습관화 설계**: 매일 10분 읽기 스트릭·알림 등으로 매일 성경 읽기를 습관으로 만든다.
- **무료 데이터 우선**: 상업적 제약이 적은 무료 API/데이터를 우선 검토.
- **웹·앱 동시 지원**: 한 번의 백엔드로 웹과 Flutter 앱 모두 지원.

## 3. 목표 / 비목표 (Goals / Non-Goals)

### 목표 (Goals)

- **지금 내 상황에 맞는 말씀**을 감정·상황 테마로 추천한다.
- 매일 **1장(Chapter)**을 제안하고, AI가 **쉬운 설명 + 삶 적용 한 줄**을 제공한다.
- **읽기 스트릭(10분 기준)**으로 매일 성경 읽기 습관을 형성한다.
- 사용자가 **검색**(책명·장·절 참조·키워드)으로 원하는 말씀을 찾을 수 있게 한다.
- **전체 리스트**(책→장→절)로 성경 구조를 탐색할 수 있게 한다.
- **웹**과 **Flutter 앱**에서 동일한 기능을 제공한다.

### 비목표 (Non-Goals)

- 교리·신학적 "해석"을 AI가 권위 있게 제시하는 기능은 하지 않는다. (쉬운 설명 + 삶 적용만)
- 초기에는 **오프라인 전용** 모드는 목표로 두지 않는다(필요 시 후순위).
- 다른 QT/묵상 앱과의 심층 연동(예: YouVersion 동기화)은 후순위.

## 6. 인증 / 로그인

### 6.1 소셜 로그인 제공자

| 플랫폼 | 구글 로그인 | 카카오 로그인 | 애플 로그인 |
|--------|:-----------:|:------------:|:-----------:|
| **웹** | O | O | — |
| **Android** | O | O | — |
| **iOS** | O | O | O |

- iOS는 Apple 앱스토어 정책상 소셜 로그인 제공 시 **애플 로그인 필수**.
- 웹/Android는 구글 + 카카오 2종으로 시작.

### 6.2 인증 흐름

- **OAuth 2.0** 기반. 소셜 로그인 후 자체 JWT 토큰 발급.
- 최초 로그인 시 자동 회원가입 (별도 회원가입 폼 없음).
- 로그인 상태 유지: 웹은 httpOnly 쿠키 또는 localStorage, 앱은 Secure Storage.

### 6.3 로그인 연동 기능

- **즐겨찾기 서버 동기화**: 로그인 사용자는 즐겨찾기를 서버 DB에 저장 → 기기 간 동기화.
- **최근 읽은 장 동기화**: 로그인 시 서버에 저장, 비로그인 시 로컬 저장 유지.
- **읽기 스트릭 동기화**: 로그인 시 서버에 기록, 비로그인 시 로컬 저장.
- **비로그인 사용 가능**: 로그인 없이도 앱의 모든 기능 사용 가능 (로컬 저장). 로그인은 동기화를 위한 선택 사항.

### 6.4 기술 구현

| 항목 | 선택 |
|------|------|
| **구글 로그인** | Firebase Auth 또는 Google OAuth 2.0 직접 연동. |
| **카카오 로그인** | Kakao SDK (웹: JavaScript SDK, Flutter: kakao_flutter_sdk). |
| **애플 로그인** | Sign in with Apple (Flutter: sign_in_with_apple 패키지). |
| **백엔드** | 소셜 토큰 검증 → 자체 JWT 발급. 사용자 테이블에 provider + provider_id 저장. |

## 7. UI 구조: 3탭 네비게이션

앱(웹·Flutter)은 하단 탭 3개로 구성한다.

### 탭 1: 홈

홈 화면은 위에서 아래로 스크롤하며 다음 섹션들을 순서대로 표시한다.

#### ① 검색 바
- 홈 최상단 고정. 참조(예: "요한복음 3장") 또는 키워드 입력 시 바로 검색 결과로 이동.
- 별도 검색 탭 없이 홈에서 바로 접근 가능하게 한다.

#### ② 읽기 스트릭 배너
- 현재 연속 읽기 일수 표시 (예: "7일 연속 말씀 읽는 중!").
- 오늘 읽기 진행 상황: 프로그레스 바 (예: "오늘 6분 / 10분").
- 오늘 아직 안 읽었으면 "오늘의 말씀을 읽어보세요" 유도 문구.
- 시각적으로 불꽃/달력 아이콘 등으로 동기부여.

#### ③ 오늘의 말씀 카드
- 날짜 기반으로 결정된 **오늘의 1장(Chapter)**을 큰 카드로 강조 표시.
- 참조(예: "요한복음 3장") + 핵심 구절 미리보기(대표 1절) + AI 쉬운 설명 미리보기(1줄).
- 카드 탭 시 → 장 읽기 화면(장 전체 본문 + AI 쉬운 설명 + **삶 적용 한 줄** + 즐겨찾기 버튼).

#### ④ 감정·상황 맞춤 추천 (핵심 차별 기능)
- **"지금 어떤 마음인가요?"** 문구와 함께 감정·상황 칩(Chip) 가로 스크롤.
- 테마 예시: "위로가 필요해", "감사해", "불안해", "용기가 필요해", "사랑", "지혜", "외로워", "화가 나" 등.
- 칩 선택 → 해당 감정/상황에 맞는 **핵심 구절 카드 리스트** 표시 (대표 구절로 진입).
- 카드 탭 시 → 해당 구절이 속한 **장 전체 읽기** + AI 쉬운 설명 + 삶 적용 한 줄.

#### ⑤ 구약 추천
- 구약에서 추천하는 장 카드 리스트 (가로 스크롤 또는 세로 리스트).
- 매일 또는 주기적으로 갱신.

#### ⑥ 신약 추천
- 신약에서 추천하는 장 카드 리스트 (가로 스크롤 또는 세로 리스트).
- 매일 또는 주기적으로 갱신.

#### ⑦ 최근 읽은 장
- 사용자가 최근에 읽은 장 히스토리 (최대 10~20개).
- 가로 스크롤 카드로 표시. 탭 시 해당 장 읽기 화면으로 이동.
- 저장: localStorage(웹) / SharedPreferences(Flutter).

#### ⑧ 오늘의 한 줄 기도
- 오늘의 말씀(장)을 기반으로 AI가 **짧은 기도문 1~2문장**을 생성.
- 하단에 카드 형태로 배치. "오늘의 말씀과 함께 기도해 보세요" 문구.
- 캐시: 오늘의 말씀과 동일하게 날짜별 캐시.

### 탭 2: 즐겨찾기

- 사용자가 하트/북마크한 장 또는 구절의 리스트.
- **비로그인**: localStorage(웹) / SharedPreferences(Flutter)에 로컬 저장.
- **로그인**: 서버 DB에 저장 → 기기 간 동기화.
- 탭 시 → 해당 장 읽기 + 쉬운 설명 + 삶 적용 다시 보기.
- 즐겨찾기 해제(삭제) 가능.

### 탭 3: 설정

- **로그인/로그아웃**: 소셜 로그인 버튼 (구글/카카오/애플) 또는 로그아웃.
- **읽기 스트릭 기록**: 연속 읽기 통계, 월간 달력 뷰 (읽은 날 표시).
- 번역본 선택 (영어: KJV / WEB 등).
- AI 설명 언어 설정 (한국어 / 영어).
- 다크 모드 ON/OFF.
- 앱 정보 / 버전 / 라이선스.

## 8. 기능 요구사항

### 8.1 오늘의 말씀 (Daily Chapter)

- **입력**: 날짜(오늘) + 선택: 구약만 / 신약만 / 번갈아 등.
- **출력**: 그날 정해진 **1장(Chapter)**의 전체 본문(책명, 장 번호, 절 전체).
- **로직**: 날짜를 시드로 사용해 **결정적**으로 한 장 선택(같은 날짜면 모든 사용자에게 동일 장). 성경 전체 1,189장을 순차 또는 큐레이션 순서로 배정.
- **표시**: 장 전체 본문 + 참조(예: 요한복음 3장) + 번역본 표기.
- **스트릭 연동**: 이 화면에서의 체류 시간이 읽기 스트릭에 합산됨.

### 8.2 AI 쉬운 설명 + 삶 적용 (핵심 차별 기능)

- **역할**: 해당 **장 전체**에 대해 두 가지를 제공:
  1. **쉬운 설명** — 장의 핵심 내용 요약, 등장인물·배경·문맥 풀어주기, 어려운 구절 짚어주기. 2~3문단.
  2. **삶 적용 한 줄** — "오늘 이렇게 적용해 보세요: ___" 형태의 실천 가능한 한 문장.
- **구현**: 장 전체 본문(또는 요약)을 LLM API에 넘기고 아래와 같은 프롬프트 사용:
  > "이 성경 한 장의 내용을 초등학생도 이해할 수 있게 쉬운 말로 요약·설명해 줘. 핵심 메시지, 등장인물, 배경을 포함해. 교리나 해석은 넣지 마.  
  > 마지막에 '오늘 이렇게 적용해 보세요:'로 시작하는 실천 가능한 한 문장을 추가해 줘."
- **캐시**: 같은 장에 대한 설명은 저장해 두고 재사용하면 비용·속도 유리.

### 8.3 감정·상황 맞춤 추천 (핵심 차별 기능)

- **컨셉**: 사용자가 책/장/절이 아니라 **"지금 내 감정/상황"**에서 출발하여 말씀을 만난다.
- **테마 목록**: "위로가 필요해", "감사해", "불안해", "용기가 필요해", "사랑", "지혜", "외로워", "화가 나", "기쁨", "평안" 등 10~15개.
- **데이터**: 테마별로 **대표 구절 + 해당 구절이 속한 장** 정보를 사전 큐레이션(JSON). 테마당 10~20개.
- **표시**: 홈에서 "지금 어떤 마음인가요?" + 가로 스크롤 칩(Chip) → 선택 시 핵심 구절 카드 리스트.
- **상세**: 카드 탭 시 → 해당 구절이 속한 **장 전체 읽기** + AI 쉬운 설명 + **삶 적용 한 줄** + 즐겨찾기 버튼. 대표 구절은 하이라이트 표시.

### 8.4 읽기 스트릭 (Reading Streak)

- **목적**: 매일 10분 이상 말씀을 읽는 습관을 만들어 **재방문율**을 높인다.
- **인정 기준**: 읽기 화면(장 본문, AI 설명 화면)에서 **누적 체류 시간 10분 이상** → 해당 날짜 "읽음" 처리.
  - 홈/설정/검색 등 비읽기 화면은 시간 합산에서 제외.
  - 10분 ≈ 1장 본문 읽기(~4분) + AI 설명 읽기(~2분) + 추가 탐색(~4분).
- **표시**:
  - 홈 상단 배너: 현재 연속 일수 + 오늘 프로그레스 바 (6분/10분).
  - 설정 탭: 월간 달력 뷰 (읽은 날 표시).
- **등급**:
  - 기본: 10분 달성 → 불꽃 1개 (스트릭 인정).
  - 열심: 20분 이상 → 불꽃 2개 (또는 금색 불꽃).
- **저장소**:
  - **비로그인**: localStorage / SharedPreferences.
  - **로그인**: 서버 DB에 저장 → 기기 간 동기화.
- **알림 (Phase 2)**: 매일 정해진 시간에 푸시 알림 → "오늘의 말씀이 도착했어요".

### 8.5 즐겨찾기 (Favorites)

- **동작**: 장 읽기 화면에서 하트/북마크 아이콘 탭 → 즐겨찾기 저장.
- **저장 데이터**: 장 참조(책·장), 대표 구절 텍스트, 저장 일시.
- **저장소**:
  - **비로그인**: 클라이언트 로컬 저장(localStorage / SharedPreferences).
  - **로그인**: 서버 DB에 저장 → 기기 간 동기화. 로그인 시 로컬 데이터를 서버로 병합.
- **목록**: 즐겨찾기 탭에서 저장순(최신 위) 리스트 표시.
- **삭제**: 스와이프 또는 버튼으로 즐겨찾기 해제.

### 8.6 최근 읽은 장 (Recently Read)

- **동작**: 사용자가 장 읽기 화면을 열 때마다 히스토리에 자동 저장.
- **저장 데이터**: 장 참조(책·장), 열람 일시, 읽기 진행률(선택).
- **저장소**:
  - **비로그인**: localStorage(웹) / SharedPreferences(Flutter). 최대 20개, 초과 시 가장 오래된 항목 삭제.
  - **로그인**: 서버 DB에 저장 → 기기 간 동기화.
- **표시**: 홈에서 가로 스크롤 카드 리스트. 탭 시 해당 장 읽기 화면으로 이동.

### 8.7 오늘의 한 줄 기도 (Daily Prayer)

- **역할**: 오늘의 말씀(장)을 기반으로 AI가 **짧은 기도문(1~2문장)**을 생성.
- **구현**: 오늘의 말씀 본문을 LLM API에 넘기고, "이 장의 핵심 메시지를 바탕으로 짧고 따뜻한 기도문 1~2문장을 써 달라"는 프롬프트 사용.
- **캐시**: 오늘의 말씀과 동일하게 날짜별 캐시 (하루 1회 생성).
- **표시**: 홈 하단 카드. "오늘의 말씀과 함께 기도해 보세요" 안내 문구 포함.

### 8.8 검색 (Search)

- **검색 대상**:  
  - 책명·장 참조(예: "요한복음 3장", "시편 23")  
  - (가능하면) 본문/키워드 검색
- **동작**: 검색어 입력 → 후보 장/구절 목록 → 선택 시 해당 장 전체 읽기 + 쉬운 설명 + 삶 적용 표시.
- **제약**: 사용 중인 성경 API가 키워드 검색을 지원하는지에 따라 1차는 "참조 검색"만, 2차에 본문 검색 추가 가능.

### 8.9 전체 리스트 (Books → Chapters)

- **1단계**: 성경 **책 목록** (구약 39권 + 신약 27권 등).
- **2단계**: 책 선택 시 **장 목록**.
- **3단계**: 장 선택 시 → **장 전체 읽기 화면** (본문 + AI 쉬운 설명 + 삶 적용).
- **목적**: 성경 구조를 탐색하면서 원하는 장을 골라 읽기.  
  "오늘의 말씀"만이 아니라, 원하면 아무 장이나 골라 읽을 수 있어야 함.

## 9. 성경 데이터: 무료 API 조사 결과

### 9.1 bible-api.com (무료, API 키 불필요)

| 항목 | 내용 |
|------|------|
| **URL** | `https://bible-api.com/` |
| **인증** | 없음 |
| **제한** | 15 requests / 30초 (IP 기준). 전체 성경 일괄 다운로드 금지. |
| **번역** | WEB(기본), KJV, ASV, CUV(중국어) 등. **한국어 번역 없음.** |
| **전체 리스트** | 가능. `/data` → 번역별 `/data/{translation}` → `/data/{translation}/{BOOK_ID}` → `/data/{translation}/{BOOK_ID}/{CHAPTER}` 로 책·장 탐색. |
| **장 조회** | `https://bible-api.com/john+3` 형태로 장 전체 본문 반환. |
| **랜덤** | `/data/{translation}/random` 등 지원. |

**정리**:  
- 장 단위 조회가 가능하므로 **1장 읽기** 기능에 적합.  
- **한국어**는 이 API만으로는 불가 → 별도 한글 소스 필요.

### 9.2 API.Bible (api.scripture.api.bible)

| 항목 | 내용 |
|------|------|
| **인증** | API 키 필요 (회원가입 후 발급) |
| **무료** | Starter: 비영리, 월 5,000 호출, 선택 3개 저작권 번역 + CC/PD 성경. |
| **언어** | 다국어(ISO 639-3). 한국어 버전 존재 여부는 대시보드에서 확인 필요. |
| **목록/검색** | 책·장·절 목록 API, 검색 API, 장 전체 본문 API 제공. |

**정리**:  
- **한국어**를 쓸 경우 API.Bible 한국어 버전 확인 후 사용 검토.  
- 장 단위 조회 지원. 무료 한도 내에서 운영 가능.

### 9.3 기타

- **한국성경공회 API**: 한글 성경 검색/조회. 상업 사용·조건은 별도 확인 필요.
- **자체 데이터**: 공개 한글 성경 텍스트(라이선스 확인 후)를 JSON 등으로 보유하고, 서버에서 제공하는 방식 가능.

### 9.4 PRD 기준 제안

| 단계 | 언어 | 데이터 소스 |
|------|------|-------------|
| **MVP** | 영어 우선 | bible-api.com으로 장 단위 조회 + 전체 리스트 + 검색. |
| **MVP** | 한국어 추가 시 | API.Bible(한국어 확인) 또는 한국성경공회 API 또는 자체 JSON. |
| **이후** | 다국어/다번역 | API.Bible 확장 또는 추가 API/데이터. |

## 10. AI(LLM) API: 무료 티어 조사 결과

"장 단위 쉬운 설명 + 삶 적용" 및 "오늘의 한 줄 기도" 생성에 사용할 LLM API. MVP는 **무료 티어**로 시작한다.

### 10.1 Google Gemini API (AI Studio) — 권장

| 항목 | 내용 |
|------|------|
| **URL** | `https://ai.google.dev/` |
| **인증** | API 키 (Google AI Studio에서 발급, 카드 불필요) |
| **무료 한도** | Gemini 2.5 Flash: 10 RPM, 250 RPD, 250K TPM. Flash-Lite: 15 RPM, 1,000 RPD. |
| **장점** | 긴 컨텍스트(1M), 한국어 품질 우수, 장 전체 본문 입력에 충분. |
| **주의** | 무료 티어 데이터는 Google 제품 개선에 사용될 수 있음. 일일 쿼터 리셋은 자정(PT). |

### 10.2 OpenRouter (무료 모델)

| 항목 | 내용 |
|------|------|
| **URL** | `https://openrouter.ai/` |
| **인증** | API 키 (회원가입, 카드 불필요) |
| **무료 한도** | `:free` 모델: 20 RPM, 200 RPD. Llama 3.3 70B, Qwen3 80B, Gemma 3 27B 등 28개 모델. |
| **장점** | 모델 교체 자유로움(벤더 락인 없음). OpenAI 호환 API 형식. |
| **주의** | 무료 모델 품질·속도 편차 있음. 일부 모델 한국어 약함. |

### 10.3 Groq (무료 티어)

| 항목 | 내용 |
|------|------|
| **URL** | `https://console.groq.com/` |
| **인증** | API 키 (회원가입, 카드 불필요) |
| **무료 한도** | llama-3.3-70b: 30 RPM, 12K TPM, 1K RPD. llama-3.1-8b: 30 RPM, 6K TPM, 14.4K RPD. |
| **장점** | 응답 속도 매우 빠름 (LPU 추론). |
| **주의** | 모델 선택지 제한적. 한국어 성능은 Gemini 대비 약함. |

### 10.4 AI API 선정 제안

| 우선순위 | API | 용도 |
|----------|-----|------|
| **1순위 (MVP)** | **Google Gemini Free** | 장 단위 쉬운 설명 + 삶 적용 품질이 가장 좋고, 무료 한도 충분. |
| **2순위 (대체)** | **OpenRouter Free** | Gemini 한도 초과 시 폴백 또는 영어 전용 설명. |
| **3순위 (속도)** | **Groq Free** | 빠른 응답이 필요한 경우 보조 사용. |

- **캐시 전략**: 동일 장 설명은 DB/파일에 캐시하여 API 호출 최소화 → 무료 한도 내 운영 가능. 성경 전체 1,189장이므로 전부 캐시해도 충분.
- **폴백**: 1순위 API 한도 초과 시 자동으로 2순위로 전환하는 로직 권장.

## 12. MVP 범위 (Phase 1)

- **웹**에서 다음까지 구현하는 것을 MVP로 제안.
  - **홈 화면**: 검색 바 + 읽기 스트릭 배너(프로그레스 바) + 오늘의 말씀 카드 + 감정·상황 맞춤 추천 + 구약/신약 추천 + 최근 읽은 장 + 오늘의 한 줄 기도.
  - **오늘의 말씀**: 날짜 기준 1장(Chapter). 장 전체 본문 표시.
  - **AI 쉬운 설명 + 삶 적용**: 장마다 AI가 쉬운 설명(2~3문단) + "오늘 이렇게 적용해 보세요" 한 줄 제공(캐시 권장).
  - **감정·상황 맞춤 추천**: "지금 어떤 마음인가요?" 칩 → 핵심 구절로 진입 → 해당 장 전체 읽기.
  - **읽기 스트릭**: 읽기 화면 체류 10분 기준. 연속 일수 기록 + 홈 프로그레스 바 표시.
  - **오늘의 한 줄 기도**: 오늘의 말씀(장) 기반 AI 기도문(캐시 권장).
  - **즐겨찾기**: 장 저장/삭제, 즐겨찾기 탭에서 리스트 표시.
  - **최근 읽은 장**: 열람 히스토리 자동 저장, 홈에서 표시.
  - **로그인**: 구글/카카오 소셜 로그인 → 즐겨찾기·최근 읽은 장·스트릭 서버 동기화.
  - **검색**: "책 장" 형태 참조 검색 → 결과 장 전체 본문 + 쉬운 설명 + 삶 적용.
  - **전체 리스트**: 책 목록 → 장 목록 → 장 읽기 화면까지 탐색 가능.
- **Flutter 앱**: 위와 동일 기능을 동일 API로 구현. iOS는 애플 로그인 추가. (MVP를 웹 먼저 한 다음 앱으로 확장해도 무방.)

### MVP 완료 정의 (DoD)

- [ ] 구글/카카오 소셜 로그인이 동작하고, 로그인 시 즐겨찾기·최근 읽은 장·스트릭이 서버에 동기화된다.
- [ ] 비로그인 상태에서도 모든 기능이 로컬 저장 기반으로 동작한다.
- [ ] 홈 화면에 8개 섹션이 표시된다.
- [ ] 날짜별 "오늘의 말씀"이 1장(Chapter) 단위로 결정되고, 장 전체 본문이 표시된다.
- [ ] 해당 장에 대한 "쉬운 설명(2~3문단) + 삶 적용 한 줄"이 AI로 생성되어 표시된다(캐시 가능).
- [ ] 해당 장에 대한 "오늘의 한 줄 기도"가 AI로 생성되어 표시된다(캐시 가능).
- [ ] 감정·상황 칩 선택 시 핵심 구절 리스트 → 해당 장 전체 읽기로 연결된다.
- [ ] 읽기 스트릭이 **10분 체류 기준**으로 기록되고, 홈 배너에 프로그레스 바 + 연속 일수가 표시된다.
- [ ] 즐겨찾기 저장/삭제가 동작하고, 즐겨찾기 탭에서 리스트가 표시된다.
- [ ] 최근 읽은 장이 자동 저장되고, 홈에서 표시된다.
- [ ] 사용자가 책·장 참조로 **검색**할 수 있고, 결과 장 본문이 표시된다.
- [ ] **전체 리스트**(책 → 장)로 탐색 후 장을 선택해 읽을 수 있다.
- [ ] 웹에서 위 기능이 동작한다.
- [ ] (선택) Flutter 앱에서 동일 기능이 동작한다.

- 오프라인 캐시.
- 성경 퀴즈.
- 구절 이미지 카드 생성 및 SNS 공유 기능.
- 다국어 확장 (스페인어, 포르투갈어 등).

## 14. 리스크 / 오픈 이슈

- **한국어 성경**: 무료·상업 이용 가능한 API/데이터 확정 필요. API.Bible 한국어, 한국성경공회, 자체 JSON 중 선택.
- **저작권**: 사용하는 번역본의 이용 조건(표기, 비영리 등) 준수.
- **AI 삶 적용**: "적용" 문구가 교리적 해석으로 오인되지 않도록 프롬프트 정교하게 설계. "~해 보세요" 수준의 부드러운 제안으로 유지.
- **AI 기도문**: 특정 교단/교리에 치우치지 않는 중립적 기도문 프롬프트 설계 필요.
- **장 단위 AI 설명**: 장 전체 본문을 LLM에 넘기면 토큰 사용량이 구절 대비 늘어남. 캐시로 1,189장 전부 커버하면 이후 추가 호출 불필요.
- **경쟁**: YouVersion·갓피플 대비 "AI 쉬운 설명 + 삶 적용 + 감정 맞춤"이 충분한 전환 동기가 되는지 초기 사용자 피드백으로 검증 필요.

## 15. 용어 정리

- **오늘의 말씀**: 날짜에 따라 결정되는 "오늘 읽을 1장(Chapter)".
- **장(Chapter)**: 성경의 기본 읽기 단위. 평균 20~30절, 약 3~5분 분량.
- **쉬운 설명**: AI가 장 전체의 핵심 내용·배경·문맥을 쉽게 풀어 쓴 텍스트(2~3문단). "해석"이 아님.
- **삶 적용 한 줄**: AI가 장을 바탕으로 "오늘 이렇게 적용해 보세요"라고 제안하는 실천 가능한 한 문장.
- **감정·상황 맞춤 추천**: "위로가 필요해", "불안해" 등 사용자의 현재 감정/상황에서 출발하여 관련 장을 추천하는 기능.
- **읽기 스트릭**: 읽기 화면에서 10분 이상 체류한 날의 연속 일수 기록. 습관화를 돕는 장치.
- **오늘의 한 줄 기도**: 오늘의 말씀(장)을 바탕으로 AI가 생성하는 짧은 기도문.
- **전체 리스트**: 성경 책 목록 → 장 목록 탐색 구조.
- **참조 검색**: "요한복음 3장"처럼 책·장으로 검색하는 방식.
